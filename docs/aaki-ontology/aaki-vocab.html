AAKI Vocabulary

Namespace: http://open-services.net/ns/aaki#  |  Preferred prefix: aaki  |  Issued: 2026-05-20  |  License: Apache 2.0

This vocabulary defines the classes and properties used to describe AI Assisted Knowledge Integration (AAKI) itself — its stages, facets, governance patterns, actors, artifacts, systems, domains, and realizations. The vocabulary contains types only; the canonical AAKI taxa (Define / Instantiate / Activate, the three Facets, the three GovernancePatterns) are A-box content in a running AAKI server's repository, not named individuals declared here.

Every AAKI class is rdfs:subClassOf oslc_am:Resource so AAKI resources inherit the common OSLC core properties (dcterms:title, dcterms:description, dcterms:creator, dcterms:created, dcterms:modified, oslc:serviceProvider, oslc:instanceShape, plus the six common link types derives / elaborates / refine / external / satisfy / trace) via OSLC ResourceShape oslc:superShape inheritance from the OASIS-standard AM Resource shape.

See also: AAKI Shapes, class diagram, stage-lifecycle sequence diagram, AAKI framework document.

Classes

Stage aaki:Stage
subclassOf oslc_am:Resource
One of the three realization stages of AAKI: Define (vocabulary and shape authoring), Instantiate (population of governed artifacts and links), or Activate (extraction of value through queries, analytics, and AI-assisted decisions). Canonical instances live in the AAKI server's repository, not in this vocabulary.
Facet aaki:Facet
subclassOf oslc_am:Resource
One of the three facets of model evolution and use that an AI assistant operates at during AAKI Activate: Tool Resource Optimization, Integration, or Analytics.
GovernancePattern aaki:GovernancePattern
subclassOf oslc_am:Resource
A named convention by which an AI assistant participates in AAKI without crossing the RACI line: Observe, Propose, or Execute.
Actor aaki:Actor
subclassOf oslc_am:Resource — abstract
Abstract supertype for participants in an AAKI lifecycle execution. Concrete subclasses are aaki:Human and aaki:AIAssistant.
Human aaki:Human
subclassOf aaki:Actor
A human participant in an AAKI lifecycle execution. SMEs, reviewers, and decision owners are all Humans. Humans remain Responsible and Accountable for every decision the system records.
AIAssistant aaki:AIAssistant
subclassOf aaki:Actor
An AI assistant participating in an AAKI lifecycle execution as a collaborator (not an agent on the RACI chart).
Decision aaki:Decision
subclassOf oslc_am:Resource
A named, attributed act by a Human that produces a state change in the governed graph. Carries provenance: who decided, when, on what evidence, in which configuration context. Makes the RACI line queryable as graph data.
Capability aaki:Capability
subclassOf oslc_am:Resource
A capability exposed by an AAKI system: Authoring, Querying, ImpactAnalysis, GapDetection, Traceability, ComplianceReporting, WhatIfAnalysis, etc.
Artifact aaki:Artifact
subclassOf oslc_am:Resource — abstract
Abstract supertype for the persistent products of AAKI: vocabularies, shapes, resource instances, links, configurations, baselines, change sets.
Vocabulary aaki:Vocabulary
subclassOf aaki:Artifact
An RDF vocabulary defining the classes and properties of a domain. Produced during AAKI Define.
ResourceShape aaki:ResourceShape
subclassOf aaki:Artifact
An OSLC ResourceShape constraining a Vocabulary for a server's API contract. Produced during AAKI Define.
ResourceInstance aaki:ResourceInstance
subclassOf aaki:Artifact
A concrete instance of a domain class, conforming to a ResourceShape and populated during AAKI Instantiate.
Link aaki:Link
subclassOf aaki:Artifact
A typed connection between two ResourceInstances, expressed as a triple in the governed graph.
Configuration aaki:Configuration
subclassOf aaki:Artifact
An OSLC Configuration that scopes which version resources are in effect for a given request.
Baseline aaki:Baseline
subclassOf aaki:Configuration
An immutable Configuration capturing a specific point-in-time state of the governed graph.
ChangeSet aaki:ChangeSet
subclassOf aaki:Artifact
An OSLC ChangeSet bundling proposed modifications to ResourceInstances and Links for review and approval as a unit.
System aaki:System
subclassOf oslc_am:Resource — abstract
Abstract supertype for the runtime systems that participate in an AAKI lifecycle execution.
OSLCServer aaki:OSLCServer
subclassOf aaki:System
An OSLC server holding the governed graph for a domain.
MCPEndpoint aaki:MCPEndpoint
subclassOf aaki:System
An MCP endpoint giving an AI assistant tool-mediated access to a System.
Datamart aaki:Datamart
subclassOf aaki:System
A materialized read-only view of the lifecycle graph, populated via OSLC TRS and queryable via SPARQL.
Tool aaki:Tool
subclassOf aaki:System
A domain-specific authoring or analysis tool that exposes its content via OSLC and/or MCP.
Domain aaki:Domain
subclassOf oslc_am:Resource
A named subject domain that AAKI is realized over — BMM, MRM, ISO 9001, etc.
Realization aaki:Realization
subclassOf oslc_am:Resource
A concrete implementation of AAKI on a specific Domain — e.g., bmm-server, mrm-server.

Properties

produces aaki:produces
A Stage produces an Artifact. Define produces Vocabulary and ResourceShape; Instantiate produces ResourceInstance and Link.
consumes aaki:consumes
A Stage consumes an Artifact produced by a prior Stage.
operatesOn aaki:operatesOn
A Stage operates on the populated graph during execution.
hasFacet aaki:hasFacet
The Activate Stage has three facets: Tool Resource Optimization, Integration, and Analytics.
governs aaki:governs
A GovernancePattern governs an AIAssistant's actions of a particular category.
owns aaki:owns
A Human owns a Decision — the RACI Responsible/Accountable position recorded as ownership in the governed graph.
assists aaki:assists
An AIAssistant assists a Stage (or an Actor in that Stage).
realizes aaki:realizes
A Realization realizes one or more AAKI Stages on a specific Domain.
exposes aaki:exposes
A System exposes Capabilities to participating AIAssistants and Humans.
scopedTo aaki:scopedTo
Identifies the System or Facet within which an AIAssistant operates for a particular interaction.
supports aaki:supports
A Capability supports one or more kinds of human Decision.
hasDomain aaki:hasDomain
A Realization has a Domain.
resolves aaki:resolves
A Configuration resolves to a set of ResourceInstance versions for a given request.
demonstratedBy aaki:demonstratedBy
A Stage is demonstrated by a Realization that exercises the Stage end-to-end.
decidedBy aaki:decidedBy
A Decision was made by a Human.
decidedAt aaki:decidedAt
The xsd:dateTime at which a Decision was made.
basedOnEvidence aaki:basedOnEvidence
An Artifact (typically an AI-produced analysis, query result, or proposal) that informed a Decision.
inConfiguration aaki:inConfiguration
The Configuration context in which a Decision was made.
effectsChange aaki:effectsChange
An Artifact whose state changed as a result of a Decision.
